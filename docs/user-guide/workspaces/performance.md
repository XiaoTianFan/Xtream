# Performance Workspace

Performance is a planned live execution and monitoring surface.

In the current runtime, use [Patch](patch.md) for standalone static aesthetic scenes and simple routed playback. Use [Stream](stream.md) for programmed scene playback, manual branches, loops, and automation.

## Current Operator Path

Until Performance is implemented:

- Use Patch to import media, map displays, route audio, meter outputs, and run simple static scenes.
- Use Stream to run scene-based shows.
- Use Config to review readiness, diagnostics, settings, and session logs.
- Use the global footer for missing-media relink, clear solo, global audio mute, display blackout, display labels, output refresh, and theme switching.

## What Performance Is Expected To Cover

The planned Performance surface is intended for live execution and monitoring. It should eventually reduce the need to move between authoring-focused views during a show.

Until it ships, do not plan an operating workflow that depends on this surface.
