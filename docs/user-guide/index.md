# Xtream User Guide

Xtream is a desktop show-control system for media-rich rooms. Use it to import visual and audio media, create display windows, route audio through virtual outputs, and build scene-based Stream playback for performances, installations, demos, and exhibitions.

This guide is written for operators, artists, venue technicians, and technically comfortable users setting up a show workstation.

## Workspaces At A Glance

| Workspace | Use it for |
| --- | --- |
| [Patch](workspaces/patch.md) | Build standalone static aesthetic scenes, run simple playback, import media, create display windows, map visuals, create virtual audio outputs, meter audio, and use the Patch transport. |
| [Stream](workspaces/stream.md) | Program scenes, triggers, audio sub-cues, visual sub-cues, control sub-cues, Flow layouts, and Gantt timing. |
| [Config and diagnostics](workspaces/config-and-diagnostics.md) | Review readiness, show settings, display composition, machine-local settings, diagnostics, and session logs. |
| [Performance](workspaces/performance.md) | Planned live execution and monitoring surface. Use Patch for static routed playback and Stream for programmed scene playback. |

## Troubleshooting And Diagnostics

Start with the status footer. It collects Patch, Stream, and shared readiness issues in one place. If media is missing, use **Relink media...** from the footer. If audio routing or displays do not behave as expected, open Config and check diagnostics before changing the show.
